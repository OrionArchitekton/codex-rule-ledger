Read the three authority boundaries correctly

1 Input must earn admission

Strict schemas, byte and count bounds, content hashes, and allowlisted structure reject malformed or unknown material. Model-bound text is redacted before transmission.

2 GPT-5.6 proposes semantics

Live mode maps complete instruction lines into typed, source-linked proposals. It uses an environment-only key, store: false, no tools, and zero automatic retries.

3 TypeScript owns the verdict

Deterministic code validates digests, coverage, exact quotes, rule form, and evidence sufficiency. It alone assigns results and the canonical SHA-256 export.

Subjective language is declined. Ambiguity routes to human review. NOT_EVIDENCED is neither failure nor compliance.

The hardest design calls
01

Admissibility before pass or fail

A diff shows what changed, not which layered instructions governed the captured run or whether its evidence supports a completion claim.

02

Missing evidence gets its own state

Treating absence as compliance admits an unverified "done." Treating it as failure rejects work that may have happened outside the supplied capture.

03

Retries change what failure means

Case 001 links failed typecheck plus later completion to a contradiction. Case 002 demonstrates why an exact successful retry must prevent that false contradiction.

04

Public judging stays recorded and keyless

Two disclosed synthetic cases cross the same typed analyzer seam as live mode without visitor input, persistence, a public model route, or API spend.

Honest v0.2 scope
Built
Browser explorer, read-only API, local Audit Bundle CLI, recorded and live analyzer adapters, canonical export.
Public
Exactly two repository-owned synthetic recorded cases. No login, key, upload, persistence, or public GPT endpoint.
Limit
Reports supplied evidence. Does not certify compliance, authenticate a trace, establish trusted time, normalize raw sessions, or execute captured commands.
